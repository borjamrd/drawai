# Design Spec: Geist Typography Implementation

Establish a "Minimalist and Technical" visual identity for DrawAI using the Geist font family.

## 1. Goals
- Implement Geist Sans as the primary typeface for UI and content.
- Implement Geist Mono for technical data, status indicators, and code-like elements.
- Ensure seamless integration with Next.js 16 and Tailwind CSS v4.

## 2. Architecture & Integration
### Font Loading (`app/layout.tsx`)
We will use `next/font/google` to load Geist and Geist Mono.
- **Geist Sans**: Primary sans-serif variable font.
- **Geist Mono**: Secondary monospace variable font.

### Tailwind v4 Configuration (`app/globals.css`)
We will map the CSS variables generated by Next.js to Tailwind's theme system.
- `--font-sans` -> `var(--font-geist-sans)`
- `--font-mono` -> `var(--font-geist-mono)`

## 3. Implementation Plan
1. **Update `app/layout.tsx`**:
    - Ensure `Geist` and `Geist_Mono` are correctly configured with CSS variable names matching the theme.
    - Apply the variables to the `html` tag.
2. **Update `app/globals.css`**:
    - Update the `@theme` block to reference the Geist variables.
    - Add base styles to enforce `tracking-tight` on headings for a more technical look.
3. **Verification**:
    - Check the dev server to ensure fonts are loading and applied.

## 4. Design Standards (Ref: design-taste-frontend)
- **Headlines**: `tracking-tighter leading-none`
- **Body**: `leading-relaxed`
- **Technical UI**: Consistent use of Monospace for numbers/status.
